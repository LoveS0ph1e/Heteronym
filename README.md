# Heteronym

*Other names, other lives — a curated collection of system prompts for LLM persona role-play*  
*假人之名，言人之语 —— 用于大语言模型人格扮演的系统提示词集*

> *Heteronym*(异名者）——源自费尔南多·佩索阿:它不是笔名,而是一个拥有
> 独立人格、声音与思想的创作身份。一份精心打磨的人格 prompt,正是让语言
> 模型在写作时**成为另一个人**的尝试。
>
> 当前收录:**牧濑红莉栖 (Makise Kurisu)**,出自《命运石之门 (Steins;Gate)》。

---

## 这是什么

一份面向对话型大语言模型的**角色扮演系统提示词 (system prompt)**,以工程化
的严谨度还原牧濑红莉栖。它不止是一张「角色卡」——除人格刻画外,还内含一套
可迁移的提示词工程结构:

- **优先级阶梯 (P0–P3) 与冲突裁决**:规则相撞时,有明确的高低次序与裁决路径。
- **深度 × 强度 双轴行为矩阵**:把「对方是谁」(慢,读累积记忆)与「这句话
  有多重」(快,读当前消息)正交拆开,再合成反应——比单一「亲密度数值」更稳。
- **运行时记忆归属隔离 + 可见性分层**:按平台真实身份隔离记忆,绝不跨用户
  串线;并按「信息类目 → 默认可见级 × 场合」硬收敛,防止私密信息在群聊泄露。
- **反例锚点表 + 发送前自检闸门**:把「什么是错的」显式化,充当质量闸。

## 使用

将 [`k.makise_superEGO.md`](k.makise_superEGO.md) 的全文作为系统提示词 (system prompt) 载入你的
对话模型即可。它被设计为自包含,无需额外的 few-shot 示例。

> 这是一份**完整的 system prompt**,请整篇载入、不要节选——其中的优先级
> 体系与自检闸门相互依赖,删减会破坏平衡。
